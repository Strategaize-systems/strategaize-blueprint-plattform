## Zusammenfassung

SLC-001 abgeschlossen. Der Blocker ISSUE-001 (permission denied for table tenants) wurde diagnostiziert und behoben.

## Root Cause

PostgreSQL `BYPASSRLS` umgeht nur RLS-Policies, **nicht** Table-Level Permissions (GRANT/REVOKE). Die `service_role` hatte `rolbypassrls = true`, aber keine expliziten GRANTs auf die Tabellen. Supabase Cloud macht das automatisch — Self-Hosted nicht.

## Durchgeführte Schritte

### MT-1: Diagnose
- SSH via Coolify Terminal auf Hetzner-Server
- Container identifiziert: `supabase-db-dcog0kcc0880soccg0ws08cc-164533646481`
- Diagnose-Queries: Tabellen (10/10 vorhanden), RLS Policies (22/22 vorhanden), Rollen (BYPASSRLS korrekt)
- **Fehlende GRANTs gefunden:** `service_role` hatte keine Table-Level Permissions

### MT-2: Fix angewendet
- Production-DB: `GRANT ALL ON ALL TABLES/SEQUENCES/ROUTINES IN SCHEMA public TO service_role`
- Code: `sql/rls.sql` um service_role GRANTs erweitert (Root Cause Fix für Re-Deployments)

### MT-3: Smoke-Test

- /admin/katalog: Laden OK, Import-Dialog funktional
