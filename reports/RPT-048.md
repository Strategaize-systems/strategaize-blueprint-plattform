## Findings

### Medium (fixed inline)

- **UUID/frage_id key-space mismatch** in `/src/app/api/tenant/runs/[runId]/freeform/map/route.ts`
  `v_current_answers.question_id` is a UUID, but LLM mapping output uses `frage_id` format (e.g. "F-BP-001"). Without resolution:
  - `hasExistingAnswer` in the response was always `false`
  - Existing answers context sent to LLM showed UUIDs (meaningless for mapping)
  **Fix:** Added UUID -> frage_id resolution via the already-loaded questions array. Commit: `c2d16a9`.
