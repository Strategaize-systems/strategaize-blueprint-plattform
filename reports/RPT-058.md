## Findings

### High (Fixed during QA)
- **Overlay early return unmounts FreeformChat**: The mapping/review overlay used an `if (showMappingOverlay) { return (...) }` pattern, which prevented the main workspace from rendering. This meant FreeformChat was unmounted during mapping, losing internal chat message state. **Fix**: Changed to render overlay as a sibling via Fragment (`<>...</>`) on top of the workspace. FreeformChat now stays mounted during mapping/review.

### Verified OK
- **Hooks order**: All 3 useEffects, 3 useCallbacks, ~25 useStates, and 5 useRefs are at lines 112-260, well before any conditional return (first at line 611). No Rules of Hooks violation.
- **No stubs**: No TODO, FIXME, HACK, console.log, or debugger statements found.
- **No orphaned references**: No imports or references to ModeSelector, QuestionOverview, or workspaceMode remain anywhere in src/.
- **Tab CSS hidden pattern**: All three tab containers correctly use `hidden` class toggling. Offen and Feedback tabs hide when inactive; Questionnaire tab is default-visible for admin/locked.
- **Feedback tab**: Correctly disabled with Lock icon and "Kommt bald" text.
- **i18n**: All 4 tab keys present in all 3 language files (DE/EN/NL).
- **Build**: Successful, no warnings.
- **WorkspaceTabs**: Clean component, exported type, no internal state, data-driven config.

### Low
- **Admin/locked users don't see tabs**: Tabs are hidden for admin and locked runs (`!isAdmin && !isLocked`). This is correct behavior — admins use the questionnaire view directly. No action needed.
