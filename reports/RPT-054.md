# Discovery: Blueprint Unified Tabbed Workspace (V3.3)

## Raw Idea Summary

Der Blueprint Run-Workspace soll von einem "Entweder-Oder"-Konzept (Mode Selector → separater Vollbild-Screen pro Modus) zu einem einzigen, integrierten Workspace mit drei Tabs umgebaut werden. Der User landet direkt im Workspace — kein Auswahl-Screen, kein Zwischenschritt.

## Likely Main Goal

Den Teilnehmer-Workflow von einer erzwungenen Wahl in ein flexibles, integriertes Arbeitserlebnis verwandeln. Die drei Bereiche (Offen, Frage für Frage, Feedback) sind komplementär, nicht alternativ.

## Scope — In

- Unified Workspace mit drei Tabs: Offen / Frage für Frage / Feedback
- Mode Selector Screen entfernen
- QuestionOverview als Zwischenseite entfernen
- FreeformChat in Tab "Offen" einbetten
- Sidebar im "Offen"-Tab sichtbar aber eingeklappt
- Freies Switchen zwischen Tabs
- Globales Auf-/Zuklappen aller Blöcke
- Feedback-Tab als Platzhalter (gesperrt, "Kommt bald")
- Mapping/Review als Vollbild-Schritt, Chat bleibt im Hintergrund offen
- Erst bei Accept werden Chat + Review geleert
- Style-Konsistenz nach Strategaize Style Guide v2

## Scope — Out

- Compliance-differenzierte Datenweitergabe (GF vs. Mirror)
- Feedback-Funktionalität (FEAT-036)
- Sidebar-Umbau
- API-Änderungen
- Onboarding-Content für Modi-Erklärung
- Rollentrennung GF vs. Mirror Fragensicht

## Open Questions Resolved

1. Sidebar im "Offen"-Tab: Sichtbar aber eingeklappt. User muss in "Frage für Frage" gehen um Antworten zu prüfen und per Block abzugeben.
2. Mapping-Review: Vollbild als eigenständiger Schritt. Chat bleibt im Hintergrund offen. User kann zurück zum Chat. Erst bei Accept werden beide geleert.
