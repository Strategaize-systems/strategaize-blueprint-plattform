# Architecture V3.2 — FEAT-035: Free-Form Chat mit LLM-Mapping

## Outcome Summary

Technische Architektur für FEAT-035 vollständig definiert. Alle Komponenten, Datenflüsse, API-Routen, Prompt-Strategien und Frontend-Patterns dokumentiert.

**Status:** Ready für /slice-planning.

## Architecture Summary

V3.2 erweitert den Workspace um einen Free-Form Chat Modus:

- **1 neue DB-Tabelle:** `freeform_conversations` (JSONB Messages, Status-Lifecycle, Mapping-Result)
- **3 neue API-Routen:** `freeform/chat`, `freeform/map`, `freeform/accept`
- **2 neue LLM-Prompt-Typen:** `freiform` (Gesprächsführung) + `mapping` (Zuordnung + Neutralisierung)
- **5 neue Frontend-Komponenten:** mode-selector, question-overview, freeform-chat, mapping-review, soft-limit-banner
- **0 neue Docker-Services, 0 neue externe Abhängigkeiten**

### Key Decisions

| ID | Entscheidung | Begründung |
|----|-------------|-----------|
| DEC-035 | JSONB statt separate Messages-Tabelle | Gespräche kurzlebig, immer als Ganzes geladen |
| DEC-036 | Batch-Mapping am Ende statt Echtzeit | Bessere Qualität, weniger Calls |
| DEC-037 | Neutralisierung im Prompt statt Post-Processing | Sprachliches Problem, kein technisches |
| DEC-038 | Kompakte Themen statt Frage-IDs im Freiform-Prompt | Natürlichere Gesprächsführung |
| DEC-039 | Phase-State im Workspace statt separate Routes | Konsistent mit bestehendem Pattern |

### Token-Budget

| Call-Typ | Max Input | Output |
|----------|----------|--------|
| Chat (Mirror) | ~9.300 | 512 |
| Chat (GF) | ~12.500 | 512 |
| Mapping (Mirror) | ~7.900 | ~2.000 |
| Mapping (GF) | ~13.000 | ~4.000 |

Geschätzte Kosten: ~$0.50-2.00 pro Gespräch (15 Nachrichten + Mapping).
