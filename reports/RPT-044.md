# Requirements V3.2 — FEAT-035: Free-Form Chat mit LLM-Mapping

## Outcome Summary

Requirements für FEAT-035 (Free-Form Chat mit LLM-Mapping auf strukturierte Fragen) vollständig abgeschlossen. Feature-Spec mit 10 Feature-Scopes (FS-01 bis FS-10), detaillierten Acceptance Criteria, Constraints, Risiken und Success Criteria erstellt.

**Status:** Ready für /architecture.

## Feature Summary

FEAT-035 schafft einen offenen Gesprächsmodus im Blueprint-Workspace — primär für Mirror-Teilnehmer, sekundär für Geschäftsführer. Das LLM führt ein aktives Gespräch, mappt am Ende die Erkenntnisse auf strukturierte Fragen und generiert neutralisierte Draft-Antworten. Der Teilnehmer reviewt und übernimmt die Drafts einzeln.

### 10 Feature-Scopes

| ID | Scope | Beschreibung |
|----|-------|-------------|
| FS-01 | Modus-Auswahl | Toggle zwischen Fragebogen-Modus und Free-Form Modus |
| FS-02 | Fragen-Übersicht | Block/Fragen-Liste vor Gesprächsstart sichtbar |
| FS-03 | Free-Form Chat | LLM-geführtes Gespräch mit Profil, Memory, Voice |
| FS-04 | Soft-Limit | Klare Berater-Empfehlung zum Cut bei ~30 Nachrichten |
| FS-05 | Conversation Storage | Eigene Tabelle für Gesprächsverläufe |
| FS-06 | LLM-Mapping | Batch-Zuordnung am Gesprächsende + Neutralisierungslayer |
| FS-07 | Mapping-Review UI | Übernehmen / Bearbeiten / Verwerfen pro Draft-Antwort |
| FS-08 | Antwort-Speicherung | Append als answer_submitted Events mit source: freeform |
| FS-09 | Block-Filter | Mirror: nur zugewiesene Blöcke. GF: alle 73 Fragen |
| FS-10 | i18n | DE/EN/NL für alle UI-Elemente und Prompts |

### Entschiedene Produkt-Fragen (aus Discovery)

1. Antwort-Kollision → Append
2. Mapping-Granularität → Batch am Ende
3. Gesprächslänge → Soft-Limit ~30 mit deutlicher Ansage
4. Mirror-Scope → gleicher Modus, Block-gefilterter Katalog
5. Free-Form primär Mirror-Feature
6. Neutralisierungslayer ist Kernfeature (Compliance)
7. Fragen vorher sichtbar
8. Rollentrennung tenant_member → separates Feature
